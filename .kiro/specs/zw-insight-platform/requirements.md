# 中维智营（ZW Insight）工程项目管理系统 - 需求规格

## 1. 概述

### 1.1 系统定位

中维智营是一套面向工程施工企业的全流程项目管理 SaaS 平台，覆盖从项目报备、投标、合同签订、施工过程、财务结算到项目归档的完整生命周期。系统采用多租户架构，支持 PC Web 端和移动端。

### 1.2 用户角色

| 角色 | 说明 |
|------|------|
| 平台管理员 | 管理租户、系统配置、流程定义 |
| 企业管理员 | 管理企业内部人员、权限、基础数据 |
| 项目经理 | 管理项目全生命周期，查看项目全部数据 |
| 财务人员 | 处理开票、收付款、报销、备用金等 |
| 施工人员 | 现场施工日志、质量安全检查、进度反馈 |
| 采购人员 | 材料采购、入库出库、库存管理 |
| 投标人员 | 投标报名、标书制作、开标记录 |
| 劳务管理员 | 劳务合同、班组管理、用工结算 |
| 机械管理员 | 设备台账、进出场、加油维修 |
| 行政人事 | 入离职、办公用品、车辆管理 |
| 供应商 | 外部用户，参与三方比价报价 |

### 1.3 系统边界

- PC Web 管理后台（企业端）
- PC Web 平台管理后台（运营端）
- 移动端 APP/H5（施工现场使用）
- 供应商报价门户（公开页面，无需登录）

## 2. 功能需求

### REQ-001: 多租户与用户体系

**优先级**: P0

#### 需求说明

- 平台支持多租户（多家企业）隔离运营，每个租户拥有独立的数据空间
- 租户通过"组织码"区分，登录时需输入组织码
- 租户有有效期管理，到期后系统功能受限
- 平台管理员可管理租户信息、续期、开通/关闭功能权限

#### 功能细节

1. **租户管理**：新增租户时分配组织码、设定有效期、配置可用功能模块
2. **用户类型**：支持按有效期时长（30天/90天/半年/一年）分类
3. **续期机制**：基于到期日期和当前日期中较大者，增加续期时长
4. **功能权限开通**：平台为每个租户单独配置可见菜单，客户管理平台仅显示已开通功能

#### 模块关系

- 影响所有业务模块的数据隔离
- 与【菜单管理】、【角色管理】联动

---

### REQ-002: 认证与安全

**优先级**: P0

#### 需求说明

- 支持用户名+密码+验证码登录（PC端）
- 支持组织码+用户名+密码+验证码登录（移动端）
- 登录失败锁定机制
- Token 有效期管理
- 密码修改与重置

#### 功能细节

1. **验证码**：可配置是否开启验证码
2. **登录锁定**：可配置检查周期、尝试次数、锁定时长
3. **Token有效期**：可配置 Token 过期时间
4. **密码修改**：登录用户自行修改密码
5. **密码重置**：管理员可强制重置指定用户密码；用户通过手机号验证后自助重置
6. **二次确认**：关键操作（删除合同、大额付款等）需二次密码确认

#### 模块关系

- 与【系统设置】配置联动
- 与【日志管理】记录登录日志联动

### REQ-003: 组织架构与权限管理

**优先级**: P0

#### 需求说明

- 树形组织机构管理（公司→部门）
- 岗位管理
- 人员管理（账号、角色、状态）
- 角色权限管理（功能权限+数据权限）
- 菜单管理（树形目录/菜单/按钮）

#### 功能细节

1. **机构管理**：树形结构，支持公司/部门类型，启用/停用状态，排序
2. **岗位管理**：岗位名称、编码、状态
3. **人员管理**：姓名、账号、电话、所属机构、岗位、角色、状态；支持批量导入导出、批量启停用
4. **角色管理**：角色名称、编码、状态；支持功能权限配置（勾选菜单/按钮）；支持数据权限范围设置
5. **菜单管理**：树形结构，目录/菜单/按钮三级；菜单含路由、组件、图标、排序、权重（平台管理可见/普通菜单）
6. **数据权限**：支持本人可见/本部门可见/本机构可见/指定用户可见/指定部门可见/自定义SQL六种类型
7. **字段级权限**：关键表单字段可按角色设置读/写/隐藏

#### 模块关系

- 所有业务模块的权限控制基础
- 与【项目成员管理】联动，决定项目级数据可见性

### REQ-004: 首页与工作台

#### 需求说明

- PC端首页展示待办、已办、快捷入口、公告、通知、消息中心

### REQ-005: 项目报备

**优先级**: P0

#### 需求说明

- 项目信息的起点，创建项目基本信息后进入项目生命周期
- 项目状态流转：草稿→已报备→投标中→中标→施工中→竣工→已关闭

#### 功能细节

1. **项目基本信息**：项目性质、项目类型、建设单位、项目概况、项目地址、项目联系人、签约公司
2. **是否投标**：无需投标的项目可直接设置为否，跳过投标流程
3. **项目预算**：由后续【预算管理】回写更新，非手动填写
4. **操作权限**：仅保存（草稿）、提交、编辑（仅草稿状态）、删除（仅草稿状态）、批量删除、查看
5. **自动编号**：项目编号按规则自动生成

#### 状态机

```
草稿 → 已报备（提交后）
已报备 → 投标中（投标报名后自动更新）
投标中 → 中标（开标记录确认中标后）
中标/已报备 → 施工中（施工合同登记后）
施工中 → 竣工（竣工验收通过后）
竣工 → 已关闭（项目结项审批通过后）
```

#### 模块关系

- 是所有业务模块的数据源头（几乎所有功能都需要选择项目）
- 项目预算由【目标成本编制】回写
- 项目状态由【投标管理】、【施工合同登记】、【竣工验收】、【项目结项】触发更新
- 项目的合同金额由【施工合同登记】回写
- 项目的累计产值由【产值上报】回写
- 项目的总收款由【回款登记】回写
- 项目的总支出由【付款申请】回写

### REQ-006: 投标管理

**优先级**: P1

#### 需求说明

- 管理项目从报名→标书制作→投标→开标→中标/未中标的完整投标流程
- 涉及投标费用缴纳和保证金的资金管理

#### 功能细节

1. **投标报名**：设置建设单位、招标方式、报名方式、报名日期、开标日期、投标方式、投标保证金金额；登记后自动更新项目状态为"投标中"
2. **投标任务分配**：将标书按商务、技术、经济、封标等任务节点分配负责人及完成日期
3. **投标费用缴纳**：发起申请→审批→财务付款→上传付款回单→结束
4. **保证金支付申请**：选择投标报名→填写保证金金额→审批→财务付款确认（填写付款日期）
5. **开标记录**：记录是否中标，中标则登记中标基本信息；更新【项目档案】和【投标档案】的投标状态
6. **保证金退回登记**：跟进保证金退回情况，登记退还金额
7. **人员证件**：管理职业技能证书；存在引用不允许删除
8. **公司证件**：管理公司资质证书；存在引用不允许删除

#### 资金流

```
保证金支付申请 → 财务付款 → 开标(中标/未中标) → 保证金退回登记
投标费用缴纳申请 → 财务付款 → 上传回单
```

#### 模块关系

- 投标报名更新项目状态
- 开标记录（中标）触发项目状态更新
- 保证金与【财务管理】付款关联
- 人员证件/公司证件供投标报名时引用

### REQ-007: 施工合同管理

**优先级**: P0

#### 需求说明

- 项目中标后签订施工合同，是项目收入管理的核心
- 合同金额是项目所有收入计算的基准

#### 功能细节

1. **施工合同登记**：所属项目、合同类型（登记/变更/补充协议）、合同编号（自动生成）、甲方名称、签订日期、开工日期、竣工日期、合同金额、合同条款等
2. **合同变更**：通过选择合同类型（变更/补充协议）来处理，变更后更新合同金额
3. **数据回写**：提交完成后自动更新项目档案中的合同金额
4. **合同跟踪字段**：累计变更金额、累计完成产值、累计开票金额、累计收款金额——由后续业务自动回写

#### 合同状态机

```
草稿 → 生效（审批通过）
生效 → 变更中（发起变更）
变更中 → 生效（变更审批通过）
生效 → 已结算（最终结算完成）
已结算 → 已关闭（项目结项后）
```

#### 模块关系

- 【产值上报】回写累计完成产值
- 【开票申请】回写累计开票金额
- 【回款登记】回写累计收款金额
- 【变更签证】回写累计变更金额
- 【项目最终结算】更新合同状态为已结算

---

### REQ-008: 产值与结算

**优先级**: P0

#### 需求说明

- 记录每次与甲方确认的产值，是开票和回款的依据
- 工程量清单为产值上报提供数据支持
- 最终结算确认合同终止金额

#### 功能细节

1. **工程量清单上传**：维护工程名称、规格、单位、工程量、单价、金额；支持手工维护和Excel导入
2. **产值上报**：关联项目和合同，填写本期甲方计量产值、产值确认日期；提交后自动更新施工合同的累计完成产值和项目信息中的累计产值金额
3. **项目最终结算**：填写最终结算金额、日期；上传结算附件（完工确认单、结算资料、审定案表等）

#### 业务规则

- 产值上报金额不能超过合同金额（含变更）
- 最终结算金额为项目收入的最终确认数
- 开票金额不能超过最终结算金额（已结算时）或累计产值（未结算时）

#### 模块关系

- 产值上报 → 更新施工合同累计产值 → 影响开票申请可开票金额
- 最终结算 → 确定项目最终收入 → 影响项目利润计算

---

### REQ-009: 预算管理

**优先级**: P0

#### 需求说明

- 设置项目目标成本预算，控制项目支出
- 预算分类涵盖：材料费、劳务费、机械费、专业分包费、间接费用等
- 支出类业务（采购、劳务、机械、分包、报销等）受预算控制

#### 功能细节

1. **目标成本编制**：关联项目，填写各成本类型明细（成本类型、名称、规格、单位、预算量、预算单价、预算合价、备注）；每个项目只允许一条目标成本编制；提交后回写项目信息中的项目预算金额
2. **目标成本变更**：可追加新项或调整旧项（设为0即去除）；每个项目可有多条变更记录；变更后重新计算项目预算总额
3. **预算控制配置**：超成本-禁止提交 / 超成本-仅提醒允许提交；可按项目单独设置
4. **二级科目管理**：维护成本大类下的二级科目，用于无合同的付款或报销选择

#### 预算控制逻辑

```
各成本类型预算余额 = 预算合价(编制+变更汇总) - 已签合同金额 - 已付无合同费用
当新增合同/付款时：
  if 控制方式 == "禁止提交" && 本次金额 > 预算余额:
    阻止提交
  elif 控制方式 == "仅提醒":
    弹出预算超支警告，允许继续
```

#### 模块关系

- 预算编制/变更 → 回写项目信息的预算金额
- 【采购合同】占用材料费预算
- 【劳务合同】占用劳务费预算
- 【机械合同】占用机械费预算
- 【专业分包合同】占用分包费预算
- 【项目报销】直接扣减间接费用预算
- 【材料调拨】累加/减少对应项目预算占用

### REQ-010: 采购管理

**优先级**: P1

#### 需求说明

- 创建材料采购合同，跟踪入库和结算
- 与材料库存联动

#### 功能细节

1. **新建材料合同**：所属项目、合同编号（自动生成）、甲方/乙方单位、签订日期、关联预算（材料费）、明细清单（名称、规格、单位、数量、合同量、合同价）、合同金额、付款条件
2. **材料合同结算**：选择项目→选择材料合同→依据到货入库收料量结算；展示合同金额、累计结算金额、本次结算金额

#### 业务规则

- 材料合同金额受材料费预算控制
- 结算金额不超过合同金额
- 结算完成后才能发起对应的付款申请

#### 模块关系

- 关联【材料库存-到货入库】，入库数据为结算依据
- 结算完成 → 可发起【付款申请】
- 合同金额占用【预算管理-材料费】

### REQ-011: 三方比价

**优先级**: P2

#### 需求说明

- 发起询价→供应商报价→汇总排名→确定中标→生成采购合同
- 支持公开邀请和定向邀请两种方式

#### 功能细节

1. **询价公告管理**：创建询价公告，配置邀请方式（公开/定向）、中标方式（最低价中标/综合评分）、邀请供应商、询价物料明细；状态流转：草稿→已发布→已报价→已中标→已公示
2. **供应商报价管理**：供应商专属报价入口，报价数据独立存储、权限隔离；提交后不可修改
3. **公开报价查询**：非菜单页面，公开邀请供应商免登提报报价和查询
4. **报价汇总排名**：自动汇总所有供应商报价，按中标规则计算排名；支持导出
5. **中标结果管理**：选定中标供应商，锁定报价数据，更新询价状态

#### 中标规则

```
最低价中标：按物料总报价从低到高排名
综合评分：按价格分(60%) + 供应商评价分(20%) + 交期分(20%)计算
```

#### 模块关系

- 中标后可直接生成【材料采购合同】
- 引用【供应商管理】中的供应商信息
- 供应商评价分来源于【供应商评价】

### REQ-012: 劳务管理

**优先级**: P1

#### 需求说明

- 管理劳务分包合同和自有劳务用工两种模式
- 分包模式：劳务合同→产值上报→结算→付款
- 自有劳务模式：班组管理→花名册→用工单→结算单→薪资统计
- 零星用工模式：花名册→点工单→结算单→薪资统计

#### 功能细节

**劳务分包合同：**
1. 创建合同：所属项目、合同编号、甲乙方、签订日期、关联预算、明细清单、合同金额、付款条件
2. 劳务产值上报：填写本期完成工程量，为结算提供依据
3. 劳务合同结算：依据产值上报作结算，展示累计结算金额
4. 劳务分包合同奖惩：记录奖惩类型、金额、事由

**自有劳务：**
1. 班组管理：建档、维护、启用/停用；被引用不可删除
2. 劳务花名册：实名建档，固定自有（绑定班组+项目）/临时零星（绑定项目）；身份证号唯一校验
3. 劳务用工单：记录每日出勤、工时、工价、加班费；仅草稿可编辑/删除
4. 劳务结算单：按周期/班组汇总用工单→审批→标记已结算→付款登记
5. 劳务薪资统计：汇总结算金额、支付金额、未付金额

**零星用工：**
1. 劳务点工单：同用工单逻辑
2. 点工结算单：同结算单逻辑
3. 点工薪资统计：同薪资统计逻辑

#### 模块关系

- 劳务合同金额占用【预算管理-劳务费】
- 结算完成 → 可发起【付款申请】
- 奖惩金额影响最终付款计算

### REQ-013: 材料库存

**优先级**: P1

#### 需求说明

- 管理项目级材料库存，跟踪入库、出库、退货、调拨、盘点全流程
- 库存数据自动维护，无需手动管理

#### 功能细节

1. **材料到货入库**：关联项目和采购合同→自动带出明细→填写本期入库数量→提交后：更新采购订单累计入库、更新项目材料库存（增加）、新增至采购对账/结算单；可选择"直接出库"，选是则同时生成一条领料出库单
2. **材料领料出库**：选择项目→领用日期、领用人、领用材料、领用数量→库存减少
3. **材料退货出库**：选择退货项目和采购订单→选择退货材料→仅退货/退货退款→库存减少
4. **材料调拨**：选择调出项目、调入项目、调拨材料→填写数量和单价→调出方库存减少、调入方库存增加→同时影响双方项目预算
5. **材料库存盘点**：针对已有库存材料盘点→盘点后直接修正库存数量
6. **项目材料库存**：自动汇总展示，不可手动修改

#### 库存计算

```
当前库存 = 累计入库 - 累计领用出库 - 累计退货出库 - 调出数量 + 调入数量 ± 盘点调整
```

#### 模块关系

- 入库数据作为【材料合同结算】的依据
- 调拨影响双方项目的【预算管理】数据
- 库存数据供【数据看板-库存分析】展示

### REQ-014: 机械管理

**优先级**: P1

#### 需求说明

- 管理租赁机械合同和自有机械两种模式
- 租赁模式：机械合同→使用过程记录→结算→付款
- 自有模式：台账→进出场→台班登记→工作量结算→加油→故障维修→维保记录

#### 功能细节

**租赁机械：**
1. 机械合同：所属项目、合同编号、供应商、签订日期、关联预算、明细清单（工程内容、计量单位、工程量、单价、总价、税率）、合同金额、付款条件；提交后更新机械预算和项目合同总额
2. 机械使用过程记录：记录使用量及加油金额，为结算提供依据
3. 机械合同结算：依据使用过程记录→填写本次结算的单价、计时、加油费用→汇总统计

**自有机械：**
1. 机械台账：电子档案，机械基础信息；被业务引用不可删除
2. 进场登记：仅未进场机械可登记，更新状态
3. 出场登记：仅已进场机械可登记；需先完成工作量结算
4. 台班/工作量登记：记录实际施工使用情况（台班数、工作量）；仅已进场可登记
5. 工作量结算：按项目汇总所有在场机械→一张结算单对应多条机械→结算完成后方可出场
6. 加油登记：记录加油信息，仅已进场可登记
7. 故障报修：上报故障+照片，仅已进场可报修
8. 维修派工：分配维修人员，生成维修工单
9. 维修过程：记录维修全流程，同步至维保记录
10. 维保记录：查询完整维保历史，数据来源自动汇总，不可手动修改

#### 自有机械状态机

```
未建档 → 已建档（机械台账新增）
已建档 → 已进场（进场登记）
已进场 → 已出场（出场登记，需先完成工作量结算）
已出场 → 已进场（再次进场登记）
```

#### 模块关系

- 机械合同金额占用【预算管理-机械费】
- 结算完成 → 可发起【付款申请】
- 加油费用、维修费用纳入项目机械成本统计

### REQ-016: 现场管理

### REQ-017: 财务管理

**优先级**: P0

#### 需求说明

- 管理项目收入（开票、回款）和支出（付款、报销、备用金）全流程
- 所有财务数据自动回写至项目信息和相关合同

#### 功能细节

**收入管理：**
1. **开票申请**：关联施工合同→自动带出合同金额、结算金额、已开票金额→填写发票类型、本次开票金额、发票抬头、纳税人识别号等→审批通过计入施工合同已开票金额
2. **回款登记**：关联施工合同→自动带出结算金额和未收金额→填写收款日期、金额、收款人、类型、账号→计入项目实际总收款和施工合同已收款金额

**支出管理：**
3. **收票登记**：按合同类型和供应商选择收票合同→自动带出已付款金额、已收票金额→填写本次收票金额→累加至合同已收票金额
4. **付款申请**：选择请款合同类型和供应商→选择请款合同→自动带出累计结算金额、未付金额→审批通过后本次付款金额累加至项目支出
5. **其他费用付款**：关联项目→填写付款信息→计入项目其他总付款
6. **项目报销**：关联项目→费用明细→可选冲抵备用金→直接扣减间接费用预算
7. **备用金申请**：关联项目→填写申请人、日期、金额→审批
8. **备用金归还**：选择备用金申请→自动带出待归还金额→填写归还金额→更新备用金申请归还金额
9. **个人报销**：个人发起→审批→财务付款

#### 财务数据流

```
收入链路：产值上报 → 开票申请 → 回款登记
支出链路：合同结算 → 收票登记 → 付款申请
备用金链路：备用金申请 → 使用(项目报销冲抵) → 备用金归还
```

#### 业务规则

- 开票金额 ≤ 累计产值（或最终结算金额）- 已开票金额
- 付款金额 ≤ 累计结算金额 - 已付款金额
- 备用金归还金额 ≤ 待归还余额
- 项目报销选择冲抵备用金时，自动扣减备用金余额

#### 模块关系

- 开票/回款回写【施工合同】和【项目信息】
- 付款回写各类合同已付金额和【项目信息】支出
- 报销扣减【预算管理-间接费用】
- 所有财务数据供【看板】分析

### REQ-022: 数据看板

#### 需求说明

- 以可视化图表展示企业经营数据，支持多维度分析

#### 功能细节

1. **公司经营状况分析**：
   - 项目地图分布
   - 建设合同总金额、累计结算、已收款、垫资、利润
   - 项目数量/类型/状态分布

   - 总应收、已收、未收、回款率
   - 各项目回款进度和未收款排行
   - 客户账款对比和明细

4. **供应商账款监控**：
   - 合同总额、计量金额、已付、未付、收票总额
   - 按项目/承包方筛选

   - 甘特图展示计划与实际进度
   - 多维度筛选、完成率、偏差分析

#### 关键计算公式

```
项目垫资 = 累计支出 - 累计回款

回款率 = 已收款 / 累计结算金额 × 100%
预算执行率 = 累计付款 / 预算总额 × 100%
```

#### 模块关系

- 不产生新数据

### REQ-023: 流程引擎

**优先级**: P0

#### 需求说明

## 5. 数据流核心链路

### 5.1 项目收入链路

```
项目报备 → 投标中标 → 施工合同登记(合同金额) → 变更签证(变更金额)
→ 产值上报(累计产值) → 项目最终结算(结算金额)
→ 开票申请(开票金额) → 回款登记(收款金额)
```

### 5.2 项目支出链路

```
预算编制(预算总额) → 各类合同签订(占用预算)
→ 产值上报/入库(形成结算依据) → 合同结算(结算金额)
→ 收票登记(收票金额) → 付款申请(付款金额)
```

### 5.3 项目利润计算

```
项目利润 = 项目收入(最终结算金额或累计产值) - 项目总支出(各类付款+报销)
项目垫资 = 项目总支出 - 项目总回款
```
